# スケジュール／タスク分離 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** マイ顧問／Googleカレンダーの「予定」を `itemType:'schedule'` として扱い、14日カレンダーに集約・ToDoリスト/完了カウントから分離し、タスク締切と視覚的に区別する。

**Architecture:** 取込層（`scripts/merge-external.mjs`）で `tags` に `予定` を含むアイテムを `itemType:'schedule'` に導出（既存の task 予定は再取込で再分類）。表示層（`index.html`）で schedule をタスク集計から除外し、14日カレンダーで専用チップ＋読み取り専用ポップに分岐。`parse-todo.mjs`（parser）は無変更。

**Tech Stack:** 素の ESM（Node 標準 `node:test`）＋ 単一ファイル `index.html`（インラインJS/CSS、ビルドなし）。設計トークンは `design-system/tokens.css`（`var(--…)`）。

## Global Constraints

- 値（色/余白/影/フォント/角丸/モーション）は `var(--…)` 経由のみ。**ハードコード禁止**。正本＝`03_共通基盤/AI-Claude_Codex/design-system/tokens.css`。
- `parse-todo.mjs` は**触らない**（別セッション担当）。変更対象は `scripts/merge-external.mjs` と `index.html` のみ。
- 予定判定＝**`Array.isArray(tags) && tags.includes('予定')`**（source 条件なし）。
- 既存の parse テスト（listTodo 6本＋viewWeekly 7本）と merge テスト（13本）は**緑を維持**。
- 予定は完了・編集不可（読み取り専用）。gcal 予定名の `📅 HH:MM` プレフィックスは残す。
- ブランチ `feat/schedule-task-split` で作業。コミットは日本語・末尾に `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`。

```javascript
function openScheduleDetail(id) {
  const t = tasks.find(x => x.id === id);
  if (!t) return;
  const d = t.due ? new Date(t.due + 'T00:00:00') : null;
  const dateText = d ? `${d.getMonth()+1}/${d.getDate()}（${['日','月','火','水','木','金','土'][d.getDay()]}）${formatDueText(t.due)}` : '日付なし';
  document.getElementById('sd-date').textContent = dateText;
  document.getElementById('sd-name').textContent = t.name || '';
  document.getElementById('sd-memo').textContent = t.memo || '';
  document.getElementById('schedule-detail').hidden = false;
}
function closeScheduleDetail() {
  document.getElementById('schedule-detail').hidden = true;
}
```
