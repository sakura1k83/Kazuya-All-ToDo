# スケジュール／タスク分離 設計

- 日付: 2026-07-08
- 対象: `todo-app`（index.html ＋ scripts/merge-external.mjs）
- ステータス: 設計（実装前）

## 背景・課題

マイ顧問の「予定（viewWeekly）」が、ToDo（listTodo）と同様に `itemType:'task'` で取り込まれており、**予定がタスク化**している。実データ（`-data/tasks.json`）でも、タグ `会計事務所+予定` の予定アイテムが全て `itemType:'task'`・`status:'未着手'` で入っていた（予定12件・純ToDo7件）。

予定は「いつ起きるか」を見るスケジュール（イベント）であり、完了して消す対象ではない。タスクと同じ扱いだと、ToDoリストと未完了カウントを予定が汚す。

## 要件（合意済み）

1. **予定＝14日カレンダー帯に集約**して時系列で見る。**ToDoリストには出さない**。
2. **予定に完了操作なし**（イベントは過ぎていくだけ）。タスクは従来通りToDoリスト＋完了。
3. カレンダー上で**予定とタスク締切を視覚的に分離**する。
4. データモデルは **A案：`itemType:'schedule'` を merge で `予定` タグから導出**（parser は無変更）。**対象は `tags` に `予定` を含む全アイテム**（マイ顧問 viewWeekly ＋ Google カレンダー。gcal 予定も tag `予定` を持つため同じく schedule 化される。gmail は tag `メール` のため対象外）。
5. 予定チップのクリック挙動は **案a：読み取り専用の詳細ポップ**（名前・日付・メモ表示のみ。完了/編集なし＝同期で上書きされるため）。

## 設計

### 1. データモデル

- タスクの `itemType` に `'schedule'` を追加（既存は `'task'` / `'project'`）。
- 予定判定＝**`tags` に `'予定'` を含む**こと（parser が既に付与）。
- スケジュールは完了概念を持たない。`status` フィールドは保持するが、表示・カウントには用いない。

### 2. 取込（scripts/merge-external.mjs）

- **新規アイテム生成時**（現 L44-54）: `tags.includes('予定')` なら `itemType:'schedule'`、でなければ従来の `'task'`。**source 条件は付けない**（gcal・mykomon 両方の `予定` タグが対象）。
  - gcal 予定は名前に `📅 HH:MM` プレフィックスが付く（merge の `PREFIX`）。これは**そのまま残す**（schedule チップ側の 📅 と重複するが無害。プレフィックス除去は本件のスコープ外）。
- **既存アイテムの再分類**（現 L58-69 の更新パス）: `tags.includes('予定')` かつ `existing.itemType !== 'schedule'` の場合、`existing.itemType = 'schedule'` に補正する。
  - これにより、既に `task` として入っている予定12件は**次回同期で自動的に schedule へ移行**（別バッチ・手動マイグレーション不要）。
- 完了スキップ判定（現 L56 `existing.completedAt || existing.status === '完了'`）はそのまま。スケジュールは完了させないため無害。
- **parse-todo.mjs（parser）は無変更**。

### 3. 表示（index.html）

タスク側の主要導線は既に `getItemType(t) === 'task'` で絞っているため、**schedule は多くが自動除外**される。手を入れるのは以下。

- **除外の穴を塞ぐ**（schedule をタスク集計に混ぜない）:
  - `renderProjectGuide`（現 L2571 `active = tasks.filter(t => t.status!=='完了')`）に `&& getItemType(t)==='task'` を追加。
  - `renderSchedules` 内のタスク集計 `activeTasks`（現 L2603）と、それを使う「2週間の対象 / 締切超過 / 期限未設定」サマリ（現 L2629-2631）に `getItemType(t)==='task'` を追加。
- **自動で正しいもの（変更不要）**: ToDoリスト active/done（現 L2749-2750）、Hero統計（stat-active/today/week/done）、検索・フィルタ結果 — すべて既に task 限定。
- **14日カレンダーの視覚分離**（現 L2601-2628）:
  - 各日のチップ生成（現 L2625）で `getItemType(t)` により分岐:
    - **schedule チップ**: 専用クラス（例 `schedule-chip--event`）。📅アイコン＋別アクセント色。優先度色・`done` 表現は使わない。onclick は**読み取り専用詳細**（下記）。
    - **task 締切チップ**: 従来の優先度色（`P_CLASS`）・`done` 表現のまま、onclick は従来の編集モーダル。
  - カレンダーの日別 `items`（現 L2613）は `due` を持つ task と schedule の両方を含める（＝両方表示・スタイルで区別）。並び順は schedule と task を混在させつつ視認できる範囲でよい（詳細は実装時 impeccable で調整）。
  - **値は全て `var(--…)` トークン経由**（色・余白・角丸・アイコン）。ハードコード禁止。正本＝`design-system/tokens.css`。
- **予定チップのクリック＝読み取り専用ポップ（案a）**:
  - 新規の軽量ポップ/ポップオーバーで「名前・日付（曜日）・メモ」を表示。完了ボタン・編集フォームは出さない。
  - 「マイ顧問と同期・自動更新」の旨を小さく注記。

### 4. 14日を超える予定

- 既存の週送りUI（`◀ 今週 ▶` / `scheduleOffset`）で前後の週を辿れる。**専用アジェンダは作らない（YAGNI）**。過去日の予定は表示窓から自然に外れる。

### 5. 見た目の最終調整

- schedule チップ・詳細ポップの視覚（色/アイコン/余白/角丸/モーション）は、**実装フェーズ冒頭で impeccable スキルを発動**して確定し、`design-system-apply` のチェックリストを完遂する（CLAUDE.md UI 3段構え）。

## テスト

- **merge-external.mjs 単体テスト（追加）**:
  - `予定` タグ付き新規 → `itemType:'schedule'`。
  - `予定` タグなし新規 → `itemType:'task'`（従来維持）。
  - 既存が `task` の `予定` アイテム → 再取込で `schedule` に再分類される。
  - schedule アイテムの name/due/memo 更新は従来契約通り効く。
- 既存の parse-todo テスト（listTodo 6本 ＋ viewWeekly 7本）は**無変更で緑維持**を確認。
- 実アプリでの目視確認: 予定がToDoリスト/未完了カウントに出ない・カレンダーで視覚分離・クリックで読み取り専用ポップ。

## 非目標（Out of Scope）

- parser（parse-todo.mjs / viewWeekly）の変更。
- 予定の編集・完了・削除機能（読み取り専用）。
- 予定専用の独立ビュー/アジェンダ画面の新設。
- `予定` タグを持たないアイテム（gmail＝tag `メール`、手動タスク等）の分類変更（それらは `task` のまま）。
  - ※Google カレンダー予定は tag `予定` を持つため**本件で schedule 化される**（非目標ではない）。既存の gcal task 予定も再取込で再分類される。
